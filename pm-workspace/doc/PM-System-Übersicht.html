Das PM-System

Von Rohdaten zum Dev-Handoff — in einem Repo

AI-Augmented PM — Woche 5

Das Problem mit einzelnen Skills

Jeder Prompt fängt bei null an.

Kein gemeinsamer Kontext.

Kein Gedächtnis. Kein Zusammenhang.

Du erklärt deinem Unternehmen jeden Tag neu wer ihr seid.

Die Idee

context/
+
input/
→
Skills
→
output/YYYY-MM-DD/

Kontext einmal befüllen.

Alle Skills lesen ihn automatisch.

Jeder Run bekommt ein eigenes Verzeichnis.

├── context/     ← Das Hirn

Was dieses System nicht ist

Kein Entscheidungsersatz
BUILD heißt: die Daten stützen diese Richtung. Du weißt was die Daten nicht wissen.

Kein Todo-System
Es produziert Artefakte — keine Deadlines, keine Zuständigkeiten.

Kein statisches Tool
Ein System das du nicht ändern kannst, kannst du nicht besitzen.

Heute anfangen

1. context/ befüllen — company.md + strategy.md

2. /system-auditor — zeigt was noch fehlt

3. Einen Flow wählen — manuell einmal durchlaufen

4. Output beurteilen — einmal korrigieren lassen

Der Rest ist Praxis.
