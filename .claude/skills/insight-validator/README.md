# InsightValidator

Insight gegen Projektkontext validieren — von First Impression bis Experiment-Design.

> Interaktiv: Checkpoint nach Schritt 1 und 2. Schritt 3+4 laufen automatisch durch.

## Input

- `context/company.md` + `context/strategy.md` (immer)
- Insight: direkt im Chat oder als Dateipfad

## Invocation

`/insight-validator`

Oder mit Insight direkt:

`/insight-validator Unsere Nutzer wollen eigentlich keine Dashboards — sie wollen Alerts.`

## Output

`output/YYYY-MM-DD/strategy/insight-validator/`

| Datei | Inhalt |
|-------|--------|
| `FirstImpression.md` | Konflikte, Ergänzungen, Lücken, Risiken |
| `UpDownSide.md` | Upside/Downside mit Web-Research |
| `HypothesisMapping.md` | Kritische Annahmen + Experiment-Design |

## Weiter mit

Experimente aus Gruppe "Zu Testen" durchführen, beginnend mit Typ 1 (Billiger Richtungscheck).
